# URL Shortener 

Консольный сервис сокращения ссылок, разработанный в рамках учебного проекта
по дисциплине **Java** на основе реального кейса от компании **PROMO IT**.

Проект реализует полный жизненный цикл короткой ссылки:
создание, использование, ограничение по количеству переходов,
автоматическое удаление по времени жизни и управление ссылками владельцем.

---

## Описание проекта

Сервис позволяет пользователю:
- сократить длинный URL и получить короткую ссылку;
- ограничить количество переходов по ссылке;
- использовать ссылку до исчерпания лимита или истечения времени жизни;
- управлять своими ссылками (просмотр, удаление, редактирование лимита).

Каждый пользователь идентифицируется автоматически с помощью **UUID**
без регистрации и авторизации.

---

## Функциональные возможности

- Создание коротких ссылок  
- Уникальные ссылки для каждого пользователя  
- Лимит переходов по ссылке  
- Ограничение времени жизни (TTL)  
- Автоматическое удаление «протухших» ссылок  
- Уведомления о недоступности ссылки  
- Управление ссылками владельцем  
- Консольный интерфейс (CLI)  
- Конфигурация параметров через файл  
- Unit-тесты бизнес-логики  
- Автоматическая сборка и тестирование через GitHub Actions

## Запуск приложения

### Требования
- Java 17  
- Maven  

### Запуск через Maven
```bash
mvn clean package
java -jar target/url-shortener-1.0-SNAPSHOT.jar
```

### Запуск через IDE
Запустить класс `Main`.

---

## Команды CLI

```
create <url>          - создать короткую ссылку
open <code>           - перейти по короткой ссылке
list                  - показать все свои ссылки
delete <code>         - удалить ссылку (только владелец)
edit <code> <limit>   - изменить лимит переходов
help                  - справка по командам
exit                  - выход из приложения
```

---

## Идентификация пользователя

- При первом создании ссылки генерируется UUID пользователя  
- UUID сохраняется в рамках текущей сессии  
- Все операции управления доступны только владельцу ссылки  

---

## Время жизни и лимиты

- Время жизни ссылки (TTL) задаётся конфигурацией  
- После истечения TTL ссылка автоматически удаляется  
- После исчерпания лимита переходов ссылка блокируется  

---

## Конфигурация

Файл: `src/main/resources/application.properties`

```properties
app.default-ttl-seconds=60
app.default-max-clicks=3
```
